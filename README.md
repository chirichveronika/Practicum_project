# Анализ продаж билетов (сервис Яндекс Афиша)

### Описание проекта:
В рамках проекта проведён исследовательский и статистический анализ данных сервиса продажи билетов (Яндекс Афиши) за период июнь — октябрь 2024 года.

Цель проекта — выявить факторы изменения спроса на мероприятия и особенности пользовательского поведения в преддверии высокого сезона (новогодних продаж).
Задачи:
  - анализ динамики ключевых метрик (заказы, выручка, пользователи)
  - исследование сезонности спроса
  - анализ популярности мероприятий, регионов и партнёров
  - сравнение поведения пользователей mobile vs desktop
  - подготовка данных и проверка продуктовых гипотез

Стек:
Python (pandas, numpy, matplotlib, seaborn, scipy)
SQL
Jupyter Notebook

В проекте используются три датасета:

1. Заказы (final_tickets_orders_df.csv)
Содержит информацию о покупках билетов:

  `order_id` — уникальный идентификатор заказа
  `user_id` — идентификатор пользователя
  `created_dt_msk`, `created_ts_msk` — дата и время заказа
  `event_id` — идентификатор события
  `cinema_circuit` — сеть кинотеатров
  `age_limit` — возрастное ограничение
  `currency_code` — валюта оплаты
  `device_type_canonical` — тип устройства (mobile / desktop)
  `revenue` — выручка
  `service_name` — оператор продаж
  `tickets_count` — количество билетов
  `total` — сумма заказа
  `days_since_prev` — дни с предыдущей покупки
  
2. События (final_tickets_events_df.csv)
Информация о мероприятиях:

  `event_id` — уникальный идентификатор мероприятия.
  `event_name` — название мероприятия. Аналог поля event_name_code из исходной базы данных.
  `event_type_description` — описание типа мероприятия.
  `event_type_main` — основной тип мероприятия: театральная постановка, концерт и так далее.
  `organizers` — организаторы мероприятия.
  `region_name` — название региона.
  `city_name` — название города.
  `venue_id` — уникальный идентификатор площадки.
  `venue_name` — название площадки.
  `venue_address` — адрес площадки.
  
3. Курсы валют (final_tickets_tenge_df.csv)
Используется для пересчёта валют:

  `data` — дата
  `nominal`, `curs` — курс
  `cdx` — код валюты

## Исследовательский анализ данных (Python, Jupyter Notebook)

**1. Загрузка и первичный анализ**
Проверка структуры данных (.info(), .describe())
Анализ пропусков и типов данных
Проверка уникальности ключевых идентификаторов

Вывод:
Данные изначально высокого качества — отсутствуют критические пропуски и дубликаты.

**2. Предобработка данных**
Были выполнены следующие шаги:

Удалены:
  - заказы без событий (~238 записей)
  - заказы с отрицательной выручкой (возвраты, ~0.13%)
  - выбросы выше 99 перцентиля (по выручке и количеству билетов)

Обработка валют: пересчёт тенге в рубли по курсу на дату и добавлен признак `revenue_rub`

Удалены неявные дубликаты по бизнес-ключу: user_id + timestamp + event_id + tickets_count + total

Результат:
Потери данных: ~1–1.5%
Итоговый датасет: ~287k записей
Данные готовы к аналитике

**3. Feature Engineering (создание признаков)**

Добавлены новые признаки:
  `one_ticket_revenue_rub` — цена одного билета
  `month` — месяц покупки
  `season` — сезон (лето / осень)

Это позволило анализировать:
  1. сезонность
  2. динамику цен
  3. поведение пользователей

**4. Исследовательский анализ (EDA)**

 Сезонность
Наблюдается рост заказов от лета к осени. Осень — пик активности пользователей

 Устройства
Основной канал — mobile. Desktop занимает значительно меньшую долю

 Типы мероприятий
Лидируют театр и концерты
Осенью присутствует рост цен на выставки, стендап, также снижение цен на концерты, театр

 Поведение пользователей
Рост заказов происходит за счёт увеличения числа пользователей (DAU)
Среднее число заказов на пользователя — стабильно
Пользователи часто покупают несколько билетов

 Недельная сезонность
Заказы распределены неравномерно
Наблюдается цикличность по дням недели

 Регионы
Высокая концентрация: ТОП-регион ≈ 30% заказов
Сильная неравномерность распределения

 Партнёры (организаторы)
Большое количество организаторов
Основная выручка — у ограниченного числа лидеров

 Статистический анализ
Гипотеза 1
H₀: mobile ≤ desktop по количеству заказов
H₁: mobile > desktop

 Результат:
p-value < 0.05 из этого следует, что гипотеза H₀ отвергнута
Осенью наблюдается рост средней цены билета для выставок и стендапа, а также снижение для концертов и театра.

Гипотеза 2
H₀: время между заказами одинаковое
H₁: у mobile больше

 Результат:
p-value < 0.05 из этого следует, что гипотеза H₀ отвергнута
Это означает, что пользователи mobile делают больше заказов, но с большим интервалом между ними.

### Итоговые выводы

- Выявлена выраженная сезонность спроса: рост активности пользователей к осени

- Поведение пользователей существенно различается в зависимости от типа устройства  
- Обнаружена высокая концентрация спроса по регионам и партнёрам  
- Структура спроса меняется в зависимости от типа мероприятия и сезона
